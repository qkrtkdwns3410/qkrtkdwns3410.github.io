# MySQL 전문 검색(Full-Text Search) - 내용 정리

> 출처: Real MySQL 8.0 - 12장 확장 검색 (12.1절)

***

## 1. 전문 검색이란?

일반적인 B-Tree 인덱스는 **짧은 단어의 동등/범위 비교**에 적합하다.\
전문 검색(Full-Text Search)은 **용량이 큰 문서를 단어 수준으로 잘게 쪼개어** 검색할 수 있게 해주는 기능이다.

* 예전 MySQL에서는 일부 스토리지 엔진(MyISAM)에서만 지원

* **MySQL 8.0부터 InnoDB 스토리지 엔진에서도 전문 검색 인덱스 지원**

***

## 2. 토큰 분리 알고리즘

MySQL 서버는 2가지 알고리즘으로 인덱싱할 토큰을 분리한다.

### 2.1 형태소 분석 (어근 분석)

* 공백 등 띄어쓰기 단위로 단어를 분리 → 조사를 제거 → 명사/어근을 추출하여 인덱싱

* **MySQL 서버에 형태소 분석 기능은 내장되어 있지 않다**

  * 단순히 공백 기준으로 토큰을 분리할 뿐

  * MeCab 라이브러리를 플러그인으로 사용할 수는 있으나, 설정이 복잡

* 서구권 언어에는 적합하지만 **한국어/중국어/일본어에는 부적합**

### 2.2 n-gram 파서

* 형태소나 어원과 **관계없이** 문자열을 **고정 길이(n)로 잘라서** 토큰을 생성하는 방식

* 형태소 분석이 필요 없으므로 **한글 문서 검색에 이용 가치가 매우 높다**

* `ngram_token_size` 시스템 변수로 n값 설정 (기본값: 2, 범위: 1~10)

| n 값   | 명칭          | 비고               |
| ----- | ----------- | ---------------- |
| 1     | uni-gram    | 토큰이 너무 작아 실용성 낮음 |
| **2** | **bi-gram** | **한글 범용 권장값**    |
| 3     | tri-gram    | 3글자 이상 검색어에 적합   |

***

## 3. n-gram 토큰화 과정

### 3.1 bi-gram (ngram_token_size=2) 토큰화 예시

문장: `"이 책은 지금까지의 매뉴얼 번역이나 단편적인 지식 수준을"`

| 문장의 단어 | 토큰(1)            | 토큰(2) | 토큰(3) | 토큰(4) |
| ------ | ---------------- | ----- | ----- | ----- |
| 이      | *(버려짐 - 길이 < n)* |       |       |       |
| 책은     | 책은               |       |       |       |
| 지금까지의  | 지금               | 금까    | 까지    | 지의    |
| 매뉴얼    | 매뉴               | 뉴얼    |       |       |
| 번역이나   | 번역               | 역이    | 이나    |       |
| 단편적인   | 단편               | 편적    | 적인    |       |
| 지식     | 지식               |       |       |       |
| 수준을    | 수준               | 준을    |       |       |
| 벗어나    | 벗어               | 어나    |       |       |
| 저자와    | 저자               | 자와    |       |       |
| 다른     | 다른               |       |       |       |
| 많은     | 많은               |       |       |       |
| MySQL  | My               | yS    | SQ    | QL    |
| 전문가의   | 전문               | 문가    | 가의    |       |

**핵심 포인트:**

* `ngram_token_size`보다 짧은 단어(예: "이")는 **버려진다**

* 단어의 **시작/중간/끝** 어디든 검색 가능 → n-gram의 강점

***

## 4. 검색어 길이와 검색 가능 여부

> **핵심 규칙: 검색어 길이 >= ngram_token_size 이면 검색 가능, 미만이면 불가**

### 4.1 bi-gram (ngram_token_size=2) 검색 결과

| 검색어      | 길이       | ngram_token_size 대비 | 결과             |
| -------- | -------- | ------------------- | -------------- |
| `"이"`    | 1글자      | **미만**              | 0건 (검색 불가)     |
| `"책"`    | 1글자      | **미만**              | 0건 (검색 불가)     |
| `"단편"`   | 2글자      | 동일                  | **1건 (검색 가능)** |
| `"적인"`   | 2글자      | 동일                  | **1건 (검색 가능)** |
| `"편적"`   | 2글자 (중간) | 동일                  | **1건 (검색 가능)** |
| `"단편적인"` | 4글자      | 초과                  | **1건 (검색 가능)** |

### 4.2 tri-gram (ngram_token_size=3) 검색 결과

| 검색어      | 길이  | ngram_token_size 대비 | 결과              |
| -------- | --- | ------------------- | --------------- |
| `"책"`    | 1글자 | **미만**              | 0건 (검색 불가)      |
| `"단편"`   | 2글자 | **미만**              | **0건 (검색 불가!)** |
| `"적인"`   | 2글자 | **미만**              | **0건 (검색 불가!)** |
| `"단편적"`  | 3글자 | 동일                  | **1건 (검색 가능)**  |
| `"편적인"`  | 3글자 | 동일                  | **1건 (검색 가능)**  |
| `"단편적인"` | 4글자 | 초과                  | **1건 (검색 가능)**  |

### 4.3 bi-gram vs tri-gram 비교 요약

```text
bi-gram:  1글자 ✗ | 2글자 ✓ | 3글자 ✓ | 4글자 ✓
tri-gram: 1글자 ✗ | 2글자 ✗ | 3글자 ✓ | 4글자 ✓
```

한글에서는 **bi-gram(n=2)이 범용적으로 적절한 선택**이다.\
단, 검색하고자 하는 검색어의 길이에 맞게 설정해야 한다.

## 5. 긴 검색어의 검색 원리

`ngram_token_size=2`인 인덱스에서 4글자 검색어 `"단편적인"`을 어떻게 찾을까?

```text
[검색 과정]

1) 검색어 "단편적인"을 ngram_token_size에 맞게 토큰으로 분리
   → "단편", "편적", "적인"

2) 각 토큰을 전문 검색 인덱스에서 동등 비교로 검색

3) 검색된 결과를 도큐먼트 ID(Document Id)로 그루핑

4) 그루핑된 결과에서 각 단어의 위치(position)를 이용해
   최종 검색어를 포함하는지 식별
```

* 전문 검색 인덱스의 **도큐먼트 ID**: 프라이머리 키와 별개로 레코드별로 부여되는 ID

* 도큐먼트 = 레코드 = 로우(Row)와 동의어

## 6. 불용어(Stopword) 설정

MySQL은 기본적으로 내장 불용어(a, the, is 등)를 사용한다.\
n-gram 전문 검색 테스트 시에는 불용어를 비활성화해야 정확한 결과를 얻을 수 있다.

```ini
# my.cnf
ft_stopword_file=''                    # 기본 파서의 불용어 비활성화
innodb_ft_enable_stopword=OFF          # InnoDB 전문 검색 엔진 불용어 비활성화
```

* `ft_stopword_file`: 기본 파서용 불용어 파일 경로 (빈 문자열 = 비활성화)

* `innodb_ft_enable_stopword`: InnoDB 전문 검색 엔진의 불용어 사용 여부 (기본값: ON)

  * **동적 변경 가능** (`SET GLOBAL innodb_ft_enable_stopword=OFF`)

  * 가능하면 `my.cnf`에 함께 설정해두는 것이 좋다

## 7. ngram_token_size 일관성의 중요성

`ngram_token_size`는 다음 **3가지 시점 모두**에 영향을 미친다:

| 시점                         | 영향              |
| -------------------------- | --------------- |
| **테이블 생성 (CREATE)**        | 전문 검색 인덱스 구조 결정 |
| **데이터 변경 (INSERT/UPDATE)** | 토큰 생성 및 인덱스 저장  |
| **검색 (SELECT)**            | 검색어 토큰 분리       |

> **한 번 전문 검색 인덱스가 생성되면&#x20;**`ngram_token_size`**를 변경하면 안 된다.**\
> 예: `ngram_token_size=2`로 테이블을 생성하고, `ngram_token_size=3`으로 쿼리하거나 INSERT하면\
> 토큰 크기가 혼합되어 검색 결과가 기대와 다르게 나온다.

### 주의사항

* `ngram_token_size`는 **읽기 전용** 시스템 변수

* `my.cnf` 설정 파일에서만 변경 가능하며, **서버 재시작 필요**

* bi-gram과 tri-gram은 **동시 사용 불가** (서버 단위 설정 → 별도 인스턴스 필요)

## 8. FULLTEXT INDEX 생성 시 주의사항

```sql
-- n-gram 파서 사용 (반드시 WITH PARSER ngram 명시)
CREATE TABLE tb_bi_gram (
    id    BIGINT NOT NULL AUTO_INCREMENT,
    title VARCHAR(100),
    body  TEXT,
    PRIMARY KEY(id),
    FULLTEXT INDEX fx_msg(title, body) WITH PARSER ngram  -- ← 핵심!
);
```

* `WITH PARSER ngram`을 **반드시** 명시해야 n-gram 파서 사용

* 생략하면 **기본 파서**(공백 구분자 기준 단어 분리)로 인덱스 생성

* 기본 파서로는 한글 검색이 제대로 동작하지 않음

***

## 9. n-gram 파서 사용 시 무시되는 시스템 변수

n-gram 파서를 사용하는 경우 다음 4개 시스템 변수는 **아무런 영향을 미치지 않는다**.\
이들은 형태소 분석 알고리즘으로 토큰을 만들어낼 때만 영향을 미친다.

| 시스템 변수                     | 설명              |
| -------------------------- | --------------- |
| `innodb_ft_min_token_size` | InnoDB 최소 토큰 크기 |
| `innodb_ft_max_token_size` | InnoDB 최대 토큰 크기 |
| `ft_min_word_len`          | 기본 파서 최소 단어 길이  |
| `ft_max_word_len`          | 기본 파서 최대 단어 길이  |

## 10. 검색 모드

### NATURAL LANGUAGE MODE

* 자연어 검색

* 관련성(relevance) 점수로 정렬

* 별도 연산자 없이 검색어만 입력

### BOOLEAN MODE

* 불리언 연산자를 사용하여 정밀한 검색 가능

| 연산자    | 의미             | 예시            |
| ------ | -------------- | ------------- |
| `+`    | 반드시 포함 (AND)   | `'+인덱스 +최적화'` |
| `-`    | 제외 (NOT)       | `'+검색 -인덱스'`  |
| *(없음)* | 포함 시 가산 (OR)   | `'인덱스 최적화'`   |
| `""`   | 구문 검색 (정확한 구문) | `'"전문 검색"'`   |
